**`Live data-story` link (A11Y/AMBER-2) — FIXED (Erick chose "make it amber", 2026-07-10).**
The link carries `class="link-live panel-live"`; `.panel-live` *did* set `--accent-on-dark`, but
`.link-live` (neutral, later in source, equal specificity) clobbered it via cascade order — so it
rendered `#e7e5e0`, not amber. That was a cascade bug, not a deliberate neutral choice. Fix: raise
specificity to `.link-live.panel-live`, so amber (`--accent-on-dark`, **8.31:1** on the dark
surface — AA-safe) wins. The Selected-work viewport now has exactly one page-amber element (the
Live link), matching the §4 amber map; the cover imagery's internal orange is content, not a page
accent token.

### Finding A11Y-3 — FIXED (Erick chose theme-aware, 2026-07-10)
The active SectionNav label (`.dot-label`) was `--text-secondary` (`#6e6a62`) → 4.28:1 on light
(marginal AA fail; only the visible active/hover label). The nav is a fixed overlay that also
passes over the dark `#projects`/`#metrics`/`#toolkit` band, where no single opaque colour clears
AA both ways. Fix: `section-nav.ts` mirrors the active section's `data-theme` onto the nav
(`.is-over-dark`), and the label + dots swap palette — label `--canvas-700` (7.35:1) on light,
`--canvas-100` (16:1) on dark; both verified. The dots follow the same flip so the nav stays
legible on the dark band (previously the light-world dot colours nearly vanished there). No pill —
Phase 0's veto stands.
